## Notes

- **SPEC-003** — Umbrella extended-slash spec; router lives in `tui/src/app/slash.rs`; sub-commands implemented under SPEC-016 (`/model`), SPEC-021 (`/mode`, `/approval`), SPEC-032 (`/skills`), SPEC-042 (`/mcp`), SPEC-062 (`/cost`), and SPEC-051..053 (resume chooser / `--resume`; `/resume` slash remains a stub toast).
- **SPEC-005** — Multiline composer via vendored `bottom_pane` / `ComposerWidget` (`tui/src/codex_adapter/composer.rs`, SPEC-106); `spec_106_shift_enter_inserts_newline` and `spec_110_p1_composer_idle` snapshot.
- **SPEC-006** — History ring and `HistoryNav` in `tui/src/app/state.rs`; no `spec_006_*` test yet.
- **SPEC-052** — `--resume <agentId>` CLI flag in `tui/src/main.rs` and `handshake()` resume path; covered indirectly by SPEC-051 chooser tests in `tui/src/app/startup.rs`.
- **SPEC-062** — `/cost` overlay and usage aggregates in `tui/src/app/overlay.rs` and `tui/src/app/usage.rs`; per-turn data exercised by `spec_060_*` / `spec_061_*` tests but no dedicated overlay snapshot test.
- **SPEC-070** — Rust TUI UI parity via cherry-picked Codex modules per `specs/20260706-codex-tui-cherry-pick.md`. `CusaViewModel` adapter (`tui/src/codex_adapter/`) maps cusa state to vendored widgets under `tui/vendor/codex-ui/`. P0–P4 wired: `custom_terminal`, `ComposerWidget`, `CodexTranscriptWidget`, tool display, status chrome with **cusa** branding.
- **SPEC-113** — Legacy `InputWidget` (`app/input.rs`) and `TranscriptWidget` rendering removed; domain types remain in `app/transcript.rs`. `draw()` composes `ComposerWidget`, `CodexTranscriptWidget`, and Codex-styled `HeaderWidget`/`StatusWidget`.
